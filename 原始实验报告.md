1.实验摘要

本项目基于 C++17 开发RV32IM 指令集软件模拟器 + 交互式调试器，属于综合型 CPU 体系结构课程设计。项目完整实现 ELF32 二进制加载、64MB 分段虚拟内存、RV32I (47 条)+M 乘除扩展 (8 条) 指令译码执行、4 类 Linux 标准系统调用；自主开发 15 条交互式调试命令，支持断点、单步、寄存器 / 内存读写。
项目采用模块化分层架构，CMake 跨平台构建，配套自动化测试脚本与官方 riscv-tests 适配套件，完成多组程序仿真验证，可用于 RISC-V 指令学习、汇编程序调试、CPU 流水线性能分析。

2.实验目的与任务要求

2.1 实验目的
深入理解 RISC-V RV32I/M 指令集编码格式、五种指令类型（R/I/S/B/U/J）；
掌握 ELF 可执行文件头部、程序段解析与虚拟内存地址映射原理；
理解加载 - 存储架构 CPU 取指、译码、执行完整流水线流程；
掌握系统调用 ecall 异常处理、堆内存动态扩容机制；
掌握交互式调试器设计思路：断点、单步、内存 / 寄存器读写；
理解五级 CPU 流水线工作机制、控制冒险冲刷、CPI 性能计算；
掌握模块化 C++ 大型工程分层开发、多人协作、自动化测试流程。

2.2 必做任务
实现 RV32I 47 条基础整数指令 + M 扩展 8 条乘除指令；
ELF32 文件解析加载，区分.text/.rodata/.data/.bss 段并分配读写执行权限；
64MB 虚拟内存模型，栈向下增长、堆通过 brk 系统调用向上扩容；
实现 write/read/exit/brk 四类 Linux 系统调用；
交互式调试器至少 15 条命令，支持断点、单步、寄存器、内存查看修改；
通过基础自动化测试，可正常运行 hello、循环、递归、数组示例程序。

6.2 各模块功能说明
（1）main.cpp 顶层入口
程序启动唯一入口，提供文本菜单：加载 ELF、进入调试 Shell、启动 GDB 服务、一键演示、退出。串联全部顶层模块，统一程序交互入口。
（2）Debugger 交互式调试器
REPL 循环读取终端输入，分词解析命令，调用 Simulator 控制 CPU 运行状态。支持断点增删、单步 N 条、连续运行、寄存器打印、内存读写、流水线性能查看、指令跟踪、模拟器重置，共 15 条标准调试指令，内置 demo 一键全流程演示功能。
（3）Simulator CPU 核心
整机控制中枢，维护模拟器全局状态（空闲 / 运行 / 暂停 / 异常 / 退出），提供 execute_one() 单步执行、run() 连续运行、reset() 重置接口。串联指令译码、内存访问、系统调用、流水线、浮点、GDB 所有底层模块，是整个系统调度核心。
（4）ELF Loader 加载器
校验 ELF 魔数、32 位架构、小端序；遍历程序头 PT_LOAD 段，根据段标识计算 MEM_READ/MEM_WRITE/MEM_EXEC 访问权限，调用内存模块映射虚拟地址，写入程序二进制数据并对 .bss 段零填充，返回程序入口 PC。
（5）riscv_isa 指令译码
定义 DecodedInst 结构体，从 32 位指令中拆分 opcode、rd、rs1、rs2、funct3、funct7；提供通用 get_bits 位截取、sign_ext 符号扩展工具函数，支撑 R/I/S/B/U/J 五类指令立即数构造、解析，为分支、跳转地址计算提供底层数据。
（6）Memory 虚拟内存
总内存容量 64MB，按 .text/.rodata/.data/.bss/ 堆 / 栈逻辑段划分地址空间；支持字节、半字、字粒度读写，每次访问校验对应段权限；提供 map_region 地址映射、mem_load/mem_store 内存读写接口，响应 brk 系统调用实现堆空间动态向上扩容。
（7）syscall 系统调用处理
识别 0x73 ecall 系统指令，读取 a0 寄存器存储的系统调用号，分别处理终端读写、程序退出、堆顶地址调整，完成模拟器与控制台 IO 交互。
（8）Pipeline 五级流水线
实现 IF 取指、ID 译码、EX 执行、MEM 访存、WB 写回五级流水；每个时钟周期各级指令向后传递；分支、跳转、ecall 异常触发流水线冲刷，统计总周期、退休指令、冲刷次数，根据 CPI = 总周期 / 退休指令 自动计算性能指标。
（9）FPU 浮点单元
实现 RISC-V F 单精度浮点扩展，包含 32 个浮点寄存器，支持浮点加、减、乘、除基础四则运算指令。
（10）GDB RSP 远程调试
通过条件编译适配 Windows / Linux 两套 Socket API，搭建跨平台套接字服务；实现 GDB RSP 协议数据包编解码，接收 GDB 下发断点、单步、内存 / 寄存器读写指令，同步控制模拟器 CPU 运行状态。

7.核心模块详细设计

7.1 RISC-V 指令译码模块

核心结构体：
 
struct DecodedInst {

    std::uint32_t opcode;  // inst[6:0]

    std::uint32_t rd;     // inst[11:7]

    std::uint32_t funct3;  // inst[14:12]

    std::uint32_t rs1;     // inst[19:15]

    std::uint32_t rs2;     // inst[24:20]

    std::uint32_t funct7;  // inst[31:25]

};

constexpr DecodedInst decode(std::uint32_t inst) {

    return DecodedInst{

        get_bits(inst, 6, 0),   // opcode

        get_bits(inst, 11, 7),   // rd

        get_bits(inst, 14, 12),  // funct3

        get_bits(inst, 19, 15),  // rs1

        get_bits(inst, 24, 20),  // rs2

        get_bits(inst, 31, 25)   // funct7

    };

}

配套工具函数get_bits截取任意比特区间，sign_ext完成 12/13/21 位立即数符号扩展，完整覆盖 U/J/B/I/S/R 五类指令格式。完整指令编码表见附录 A。

7.6 五级流水线与 CPI 统计
（1）流水线流转规则
IF (取指) → ID (译码) → EX (执行) → MEM (访存) → WB (写回)
每周期各级指令向后传递；分支、跳转、ecall 异常触发流水线 flush，清空 IF、ID 有效指令，flushes计数 + 1。
PC 更新规则：顺序执行 pc+4；分支 / 跳转使用目标地址。

（2）流水线状态转化图

        ID --> EX["EX 执行级"]
        EX --> MEM["MEM 访存级"]
        MEM --> WB["WB 写回级<br/>指令退休 retire"]
    end

    %% 周期寄存器迁移规则
    note1["每周期寄存器右移：WB←MEM、MEM←EX、EX←ID、ID←IF"]
    WB --> JUDGE{检测分支/跳转/异常？}
    JUDGE -- 无控制流变更 --> PC_SEQ["PC = PC + 4"]
    JUDGE -- 控制流变更 --> FLUSH["流水线Flush冲刷<br/>IF.valid=false, ID.valid=false<br/>flushes += 1"]
    FLUSH --> PC_BRANCH["PC = 分支偏移/跳转目标地址"]
    PC_SEQ & PC_BRANCH --> IF["生成新指令 fetch"]
